## ✨ 1. 核心特性 (Features)

- 🍏 **Apple 级品牌官网**：采用 Glassmorphism（玻璃拟态）与大留白设计，集成原生流畅的锚点平移与 AOS 滚动视差动画。
- 🛡️ **区块链哈希存证模拟**：后端基于 Node.js `crypto` 模块，提取果品核心因子（批次、农户、糖度、克重、时间戳）实时生成 SHA-256 唯一数字指纹。
- 📱 **3D 溯源数字证书**：移动端优先（Mobile-First），集成 `VanillaTilt.js` 与 `tsParticles` 粒子系统，提供极具科技感和沉浸感的“防伪证书”查询体验。
- 📊 **SaaS 级控制台**：现代化异步（AJAX）交互控制台，支持 Excel 一键无刷新上链、数据统计看板、客户留言追踪（已读/未读状态管理）。
- 🔐 **双源安全控制**：普通管理员模式与开发者超管模式（Super Admin）隔离，超管模式支持批量物理删除与强制重置。

## 🛠️ 2. 详细技术栈 (Tech Stack)

本项目采用轻量、高效的 **全栈 JavaScript** 解决方案，确保在有限的服务器资源下实现极速响应。

### 前端生态 (Frontend)
* **核心构建**：HTML5, CSS3, ES6+ JavaScript
* **UI 设计规范**：CSS Variables 动态主题, Flexbox/Grid 响应式网格系统
* **交互动画库**：
    * `AOS.js`：实现页面滚动时的组件渐显动画。
    * `VanillaTilt.js`：实现手机端溯源证书的 3D 陀螺仪倾斜与高阶反光动效。
    * `tsParticles`：渲染高性能的背景粒子光斑（模拟柑橘园光影效果）。
* **通信方案**：Native `Fetch API` (实现无刷新表单提交与数据加载)

### 后端生态 (Backend)
* **核心框架**：`Node.js`, `Express.js` (轻量级 Web 服务器框架)
* **密码学算法**：Node 内置 `crypto` (实现 SHA-256 数据加密摘要)
* **数据解析处理**：
    * `multer`：处理 `multipart/form-data`，实现前端文件安全接收。
    * `xlsx`：解析上传的 AI 分选 Excel 表格并转化为 JSON 对象。
* **持久化存储**：JSON 文件级轻量数据库 (`data.json`, `messages.json`, `config.json`)，无需繁重的 MySQL/Redis 依赖，实现“开箱即用”。

## 🧠 3. 详细设计思路 (Design Philosophy)

### 3.1 视觉交互层：去“农特产化”，迎“科技消费品化”
传统的农产品网页往往设计老旧。本项目在 UI 设计上对标一线科技产品（如 Apple）：
* **毛玻璃拟态 (Glassmorphism)**：大量使用 `backdrop-filter: blur(20px)`，在保证背景图片高清展示的同时，确保前景文字的绝对可读性。
* **微交互设计 (Micro-interactions)**：所有的按钮（Button）均设计了 `:hover` 的悬浮提亮与 `:active` 的机械下压回弹效果，大幅提升系统的“可控感”与“高级感”。

### 3.2 业务逻辑层：用数学规律建立信任
针对农产品“伪溯源、易篡改”的痛点，系统摒弃了前端随机生成假数据的做法：
* **加密闭环**：当运营人员在后台上传含有果农名称、糖度的 Excel 时，后端拦截数据流，将其拼接为混淆字符串并进行 Hash 运算。只要改动 0.1 的糖度，64 位哈希值将发生雪崩式的完全改变，从底层逻辑上还原了区块链的去中心化信任机制。

### 3.3 架构模式：绝对模块化解耦
* **CSS/JS 完全分离**：静态页面中没有任何内联逻辑，所有样式归属于 `css/` 目录，所有接口请求归属于 `js/` 目录，保证代码高可读性与可维护性。

## 🚀 5. 详细部署指南 (Deployment)

本项目采用了“环境极简”原则，去除了繁重的传统数据库（MySQL/Redis）依赖，改为轻量级 JSON 本地存储，部署极其方便，完美支持 Windows / macOS / Linux 环境。

### 环境准备
1. 确保服务器或本地电脑已安装 **Node.js** (推荐 v16.x 或以上稳定版本)。
2. 下载或 Clone 本项目到本地文件夹。

### 安装依赖包
打开终端 (Terminal) 或命令提示符 (CMD/PowerShell)，进入项目根目录，执行以下命令安装核心依赖：
```bash
# 初始化项目 (如根目录下已有 package.json 可跳过此步)
npm init -y

# 安装必需的第三方模块 (Express框架、Multer文件上传、XLSX解析)
npm install express multer xlsx
```

### 启动服务
在项目根目录下执行：
```bash
node app.js
```
成功启动后，控制台将输出以下专业提示：
```text

════════════════════════════════════════════════════════════
```

在评委答辩环节，确保你的手机（或平板）与运行后台的电脑连接在 **同一个 WiFi** 下。用手机浏览器直接访问终端中输出的 `📱 手机演示地址`（例如 `http://192.168.1.100:3000`）即可向评委展现完美的响应式移动端效果。

## 🛡️ 6. 详细运维指南 (Operations)

### 6.1 后台系统访问
* **访问地址**：`http://localhost:3000/admin.html` (本地测试) 或上述的手机演示地址。
* **默认密码**：`888888`
* **安全建议**：首次登录后，强烈建议在【⚙️ 系统设置中心】修改密码。密码状态持久化保存在 `config.json` 中。

### 6.2 客户留言状态管理 (CRM)
* 在【客户留言管理】面板中，官网新提交的表单数据默认为**未读状态**。
* 点击 **“🔴 标为已读”** 按钮，该数据条目将变灰（透明度降低），同时顶部仪表盘的“待处理留言”总数将动态 `-1`，实现完整客服闭环工作流。

### 6.3 开发者超管模式 (Super Admin)
在遇到脏数据或需要清理测试数据时，需启用超管模式：
1. 进入【系统设置中心】，找到“开发者超管权限”面板。
2. 输入超级授权码：`super999`。
3. 激活后，所有表格最前方将解锁“复选框”，上方将出现“🗑️ 批量删除”按钮，客户留言区的已读按钮也将转变为红色的强力删除按钮。
4. **注意**：超管模式下执行的删除为底层 `fs` 级别的物理擦除，不可恢复，请谨慎操作。

### 6.4 数据备份与重置 (容灾恢复)
本项目的所有状态均依赖于根目录下的 `.json` 静态文件：
* **日常备份**：直接复制项目根目录下的 `data.json`（溯源档案）和 `messages.json`（客户留言）进行异地备份即可。
* **系统重置/清库**：关闭 Node 服务，将 `data.json` 和 `messages.json` 的内容清空为 `[]`，重启服务即可恢复出厂纯净状态。
* **忘记密码怎么办**：关闭 Node 服务，将 `config.json` 的内容手动修改为 `{"admin_pwd":"新密码"}`，保存后重启即可重置密码。
